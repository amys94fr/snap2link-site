# Snap2Link — SEO audit

Goal: rank `snap2link.app` for queries that AI-tool users actually type
(*"screenshot Claude Code"*, *"screenshot to Google Drive"*, *"share
screenshot URL clipboard"*) and pass authority back to the GitHub repo.

---

## ✅ What's already in place

### Crawl & indexing

- **Static HTML** generated by Astro — every word is in the source, no
  client-side rendering blocking crawlers.
- **`robots.txt`** at `/robots.txt`: open to every crawler, points at the
  sitemap index.
- **Sitemap index** at `/sitemap-index.xml` (auto-generated by
  `@astrojs/sitemap` on every build), pointing at `/sitemap-0.xml`.
- **`<link rel="canonical">`** on the homepage — guards against
  `/index.html`, trailing slash, query string variants splitting link
  equity.
- **HTTPS** by default via Cloudflare's edge cert.
- **Mobile-friendly** responsive design, viewport meta correct.
- **`lang="en"`** on `<html>`.

### On-page

- **Title** (47 chars, under the 60-char limit Google truncates):
  *"Snap2Link — paste screenshots into Claude Code, Cursor, or any
  terminal AI"*
- **Meta description** (~150 chars, under 160):
  *"Hotkey → drag a region → public share link lands in your clipboard.
  One keystroke, on Windows, macOS, and Linux. Free, open source, MIT
  licensed."*
- **Heading hierarchy** strict — exactly one `<h1>` (the hero), every
  section is `<h2>`, every feature card title is `<h3>`. No skipped
  levels.
- **Keyword-rich body copy** that hits the target queries naturally:
  Claude Code, Cursor, Aider, ChatGPT, Continue, Zed, Tauri, screenshot,
  Google Drive, clipboard, hotkey, AppImage, .deb, .rpm, .dmg.
- **Internal anchor links** to every section (`#features`, `#how`,
  `#install`, `#open`) — improves dwell-time signals.
- **18+ outbound links** to `github.com/amys94fr/Snap2Link` with
  natural anchor text — passes authority directly to the repo.
- **`prefers-reduced-motion`** honoured — improves Core Web Vitals on
  motion-sensitive devices.

### Social / sharing

- **Open Graph tags**: type, title, description, url, image (1200×630),
  site_name, locale.
- **Twitter Card**: `summary_large_image` with title + description +
  image.
- **Theme colour** `#0F172A` for browsers that show it (mobile address
  bar, TWA splash).

### Rich results

- **JSON-LD `SoftwareApplication`** schema embedded in the head:
  - `name`, `applicationCategory`, `operatingSystem` chips
  - `softwareVersion`, `offers.price = 0`, `license` (MIT)
  - `downloadUrl` pointing at GitHub releases
  - `featureList` with the eight feature claims from the bento grid
  - `author` (Steven Abittan, GitHub URL)

  Google can use this to surface the app in *Software* rich results
  with an install button, OS chips, and (once we have ratings) stars.

### Performance

- **Astro static** — no JS framework runtime; ~3 KB of vanilla JS for
  the demo phases, install tabs, and copy buttons.
- **Geist fonts self-hosted** via Fontsource — no third-party Google
  Fonts request, no CLS from font swap.
- **Geist 700 preloaded** so the LCP (the hero headline) doesn't shift.
- **Cloudflare edge** with HTTP/3 + Brotli + Early Hints by default.
- **`preconnect` to github.com** — every outbound link to the repo
  starts the TLS handshake during idle time.
- **Tailwind 4** purged at build — total CSS payload ~25 KB gzipped.
- **`inlineStylesheets: "auto"`** — small CSS gets inlined to remove
  the render-blocking request.

## 🚧 Backlog (in priority order)

### 1. Real Open Graph image *(high impact, ~30 min)*

`og:image` currently points at `/og.png` which doesn't exist —
previews in X / Slack / Discord / iMessage will be blank or fall back
to the favicon. Need a 1200×630 PNG with:

- Lens icon + "Snap2Link" wordmark
- Tagline: *"Screenshot → public link → clipboard. One keystroke."*
- Demo screenshot (the hero animation, frozen on success state)
- Dark slate-950 background with brand-orange glow
- Drop file at `public/og.png`. No code change needed — the meta tag
  is already there.

Tools: Figma export, Canva, or `playwright screenshot`.

### 2. Submit the sitemap explicitly *(medium impact, ~5 min)*

Both:

- **Google Search Console** — add property `https://snap2link.app`,
  verify with the DNS TXT record (Cloudflare puts it in two clicks),
  submit `https://snap2link.app/sitemap-index.xml`.
- **Bing Webmaster Tools** — same flow. Bing's share is small but
  zero-cost to claim.

### 3. Backlinks from high-authority surfaces *(huge impact, recurring)*

Snap2Link's PageRank is currently zero. To climb:

- **GitHub README** — add a top-of-fold badge linking to
  `snap2link.app`. Right now the README links the other way.
- **Hacker News** — *"Show HN: Snap2Link — screenshot to public Google
  Drive link, in one keystroke (Windows/macOS/Linux)"*. Best
  posting window: Tuesday 9 AM Pacific.
- **Reddit** — `/r/rust` (Tauri tag), `/r/MacApps`, `/r/ClaudeAI`,
  `/r/cursor`. Read each subreddit's self-promo rules first.
- **X / Twitter** — single-thread launch with the demo GIF + winget
  one-liner. Tag `@cursor_ai`, `@claudeai`, `@taurirs`.
- **Awesome lists** — open PRs against
  [awesome-tauri](https://github.com/tauri-apps/awesome-tauri) and
  any *awesome-screenshot-tools* lists.
- **Product Hunt** launch — use the OG image we generate above.
- **Dev.to / Hashnode blog post** — one technical write-up
  ("How I built a screenshot tool in Tauri v2") that links the site
  in the conclusion.

These are external — out of this repo's scope but the highest-leverage
moves available.

### 4. Add `humans.txt` *(low impact, ~5 min)*

`/humans.txt` with credit + tech stack. Mostly a developer-culture
gesture, but it's another indexable file with a backlink to GitHub.

### 5. Per-section anchor titles *(low impact, ~10 min)*

Add `<a id="…">` anchors with a `title` attribute and ARIA labels —
gives Google deeplink-able section URLs (e.g. searches for
*"Snap2Link install windows"* could land on the install tab directly).

### 6. Image lazy-loading & dimensions *(low impact, ~5 min)*

The logo `<img>` in nav/footer is small but adding `width`/`height`
explicitly + `loading="eager"` for above-the-fold and `loading="lazy"`
for below-the-fold helps CLS scores.

### 7. Localised pages *(deferred — wait for traffic data)*

`/fr`, `/es` versions if Search Console shows queries from those
locales. Not worth the maintenance overhead until the data justifies
it.

### 8. Blog / Changelog page *(deferred — content cadence dependent)*

A `/changelog` or `/blog` route would expand the indexable surface.
Worth it once we have 3+ release posts to seed it.

```bash
# Sitemap accessible
curl -fsS https://snap2link.app/sitemap-index.xml | head -5

# Robots
curl -fsS https://snap2link.app/robots.txt

# Canonical + JSON-LD baked in
curl -fsS https://snap2link.app | grep -E '(canonical|application/ld\+json)'

# Lighthouse audit (run from Chrome DevTools or CLI)
npx lighthouse https://snap2link.app --view

# Rich results test
# https://search.google.com/test/rich-results?url=https%3A%2F%2Fsnap2link.app

# Mobile-friendly test
```
